# Diagnostics Style (v0.1)

Diagnostics should be dry, technical, and absurdly formal while remaining actionable.

## Requirements

- Include severity and source location when available.
- Explain what failed in plain terms.
- Add one concise hint when recovery is possible.
- Keep wording stable to support golden tests.

## Tone Examples

- `legacy module alias 'Srangs' remains supported for ecosystem continuity`
- `major transition exceeded declared compatibility comfort`
- `generic %v combobulation is tolerated but imprecise`
