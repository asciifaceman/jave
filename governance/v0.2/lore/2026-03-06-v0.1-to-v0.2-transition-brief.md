# Transition Brief: v0.1 Closeout To v0.2 Opening

Date: 2026-03-06
Author: Product and Language Engineering Joint Office
Status: Circulated for steering review

## Executive Summary

Jave v0.1 closed with stable core language behavior, deterministic diagnostics, and working cross-toolchain flow (`javec`, `baggage`, `javevm`).

The principal gap is not language novelty. The gap is practical programmability:
- no runtime I/O foundation for command-line data intake,
- no first-party file operations,
- limited standard library reach for everyday tasks,
- runtime error and exit behavior not yet contracted for automation-heavy environments.

These limitations are now the primary blockers between "toy demonstration" and "operational script and service utility" usage.

## What v0.1 Delivered

- Locked syntax and semantics with broad test coverage.
- Stable entrypoint and sequence model.
- Governance records and release process for v0.1.
- Working release automation for binaries and editor assets.

## What v0.1 Did Not Intend To Deliver

- Rich runtime system APIs.
- Filesystem primitives.
- Automation-grade error/exit contracts.
- Broad standard library utility surface.

## v0.2 Opening Priorities

Approved major candidates:
1. Runtime I/O Foundation.
2. File System Operations.
3. Richer Standard Library Surface.
4. Runtime Error and Exit Semantics.

Additional candidates queued for steering discussion:
- Path and working-directory utility set.
- Deterministic exit contract with stable failure-class mapping.

## Release-Adjacent Implications

As candidate work proceeds, documentation and examples should demonstrate:
- command-line driven workflows,
- file-driven ingestion/export,
- automation-safe failure handling,
- cross-platform path behavior.

## Recommended Governance Sequence

1. Ratify API naming boundaries for I/O and filesystem primitives.
2. Ratify runtime failure taxonomy and exit code policy.
3. Establish acceptance examples and diagnostics expectations per candidate.
4. Lock initial v0.2 scope and publish implementation milestones.
