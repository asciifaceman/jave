# Jave License (Friendly Summary)

This file is a human-friendly guide to the Jave license.

The actual legal terms are in [LICENSE](LICENSE). If anything conflicts, `LICENSE` wins.

## Quick TL;DR

You can:
- use Jave code for basically any purpose
- copy and share it
- modify it
- publish forks and derivative projects

You must:
- keep the license text with substantial portions of the code
- keep attribution notices
- clearly mark major changes
- avoid implying official endorsement

## What Is Protected

The project is open for code use, but some identity pieces are protected.

Protected areas include:
- project marks like `Jave` and `Orcal`
- continuity/story materials and official canon framing

In plain terms:
- you can fork and remix the code
- you cannot present your forked story/lore edits as official Jave canon unless approved and merged into the main project
- you cannot present your project as the official Jave distribution without permission

## The Weird Enterprise Clause

Yes, the license includes intentionally awkward corporate-style language about future clarifications and potential retroactive interpretation where law allows.

This is part of the project's satire tone.

Practical expectation from maintainers:
- community-friendly use is welcome
- good-faith experimentation, parody, and learning are fine
- deceptive branding and false "official" claims are not fine

## Warranty And Liability

Like most open licenses:
- no warranty
- use at your own risk
- maintainers are not liable for damages

## If You Are Unsure

1. Read [LICENSE](LICENSE).
2. Prefer clear attribution and clear fork identity.
3. Avoid implying your distribution is official Jave.
4. Open an issue and ask for clarification.
