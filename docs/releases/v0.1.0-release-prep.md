# v0.1.0 Release Preparation

## Scope

This checklist prepares Jave v0.1.0 for tagging and GitHub release publication.

## Technical Readiness

- [x] Full test suite passes: `go test ./... -count=1`
- [x] Version strings set to `v0.1.0` for:
- [x] `javec`
- [x] `baggage`
- [x] `javevm`
- [x] Advanced examples execute successfully
- [x] Sequence parameter support implemented and tested

## Governance Readiness

- [x] `governance/v0.1` records complete and coherent
- [x] `governance/v0.2` draft candidate scope initialized
- [x] Internal contributor workflow separated into `.github/instructions/`

## Release Automation

Workflow file:
- `.github/workflows/release-binaries.yml`

Trigger:
- On GitHub release publication (`release.published`)

Produced assets:
- `javec-<tag>-linux-amd64`
- `baggage-<tag>-linux-amd64`
- `javevm-<tag>-linux-amd64`
- `javec-<tag>-windows-amd64.exe`
- `baggage-<tag>-windows-amd64.exe`
- `javevm-<tag>-windows-amd64.exe`
- `javec-<tag>-darwin-amd64`
- `baggage-<tag>-darwin-amd64`
- `javevm-<tag>-darwin-amd64`
- `jave-vscode-install-<tag>-linux-amd64`
- `jave-vscode-install-<tag>-windows-amd64.exe`
- `jave-vscode-install-<tag>-darwin-amd64`
- `jave-vscode-extension-<tag>.tar.gz`
- `jave-vscode-extension-<tag>.zip` (when `zip` is available in workflow runtime)
- `SHA256SUMS.txt`

## Recommended Release Steps

1. Ensure default branch is green (`go test ./... -count=1`).
2. Create annotated tag:
   - `git tag -a v0.1.0 -m "Jave v0.1.0"`
   - `git push origin v0.1.0`
3. Publish GitHub release for `v0.1.0`.
4. Confirm workflow completion and attached binaries.
5. Validate checksums against `SHA256SUMS.txt`.
6. Validate downloaded binaries with `--version` on each target OS.

## Post-Release Follow-up

- Promote v0.2 candidate features to ratified scope in governance.
- Expand CI/testing for release-asset smoke execution.
- Evaluate arm64 release targets for future versions.
